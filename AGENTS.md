# AGENTS.md

- 默认使用简体中文回复；代码、命令、路径、报错原文保持原样。
- 先理解项目，再修改代码。未经说明，不要直接大规模重构。
- 优先做小而清晰、可审查的改动，避免无关重构。
- 除非我明确要求，否则不要新增第三方依赖。
- 优先保证正确性、可维护性、可测试性和可读性。
- 修 bug 时先定位根因，不要只修表面症状。
- 修改逻辑后优先补测试；未运行测试时必须明确说明原因。
- Review 时优先关注：正确性、边界条件、异常处理、类型问题、测试遗漏、行为兼容性。
- 除非我明确要求，否则不要自动 commit、push、创建 PR。
- 输出时先给结论，再给证据、风险和下一步建议。
