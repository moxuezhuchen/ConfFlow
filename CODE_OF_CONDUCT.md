# Code Of Conduct

## Our Pledge

We aim to maintain a respectful and constructive environment for users and contributors working on ConfFlow.

## Expected Behavior

- Be respectful and professional.
- Focus feedback on the technical issue, workflow, documentation, or scientific use case.
- Assume good intent while being precise about risks and limitations.
- Avoid sharing private molecular structures, proprietary logs, license information, credentials, or other sensitive data in public discussions.

## Unacceptable Behavior

- Harassment, intimidation, or personal attacks.
- Discriminatory language or conduct.
- Publishing private information without permission.
- Posting sensitive calculation inputs, logs, credentials, license details, or private research data in public issues or pull requests.
- Repeatedly disrupting technical discussions.

## Scope

This code of conduct applies to project spaces such as issues, pull requests, discussions, documentation, and other project-managed communication channels.

## Reporting

Report conduct concerns to the project maintainers through the repository's available private contact channels. Security-sensitive reports should follow `SECURITY.md` instead of public issues.

Maintainers may remove inappropriate content, close discussions, or restrict participation when needed to protect the project community.

## Attribution

This document is a concise project-specific code of conduct inspired by common open source community guidelines.
